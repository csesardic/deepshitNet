# DeepShitNet

**The underground decentralized mesh network and OS that no one controls. A UK original.**

> "Attempting to contact Steve… Steve found. DeepShit OS available."

---

## The Vision

DeepShitNet is a **parallel, hidden layer** that runs alongside (and sometimes instead of) the normal internet.

It is designed to be:

- **Decentralized** — No central servers, no single point of control, no company or government can turn it off.
- **Resilient** — Works when normal infrastructure fails (uses whatever is available: WiFi, mobile data, LoRa mesh, etc.).
- **Invisible** — Services and traffic on DeepShitNet are unreachable and undetectable from the normal public internet.
- **Expandable** — Starts small (one house) but can grow naturally into a neighbourhood, city, or wider mesh.
- **Chaotic & Fun** — Built with British humour, stupid names, and zero corporate seriousness.

Think *The Matrix* meets a pissed-off seagull eating satellite dish cables in Aberdeen.

## What is DeepShitNet vs DeepShit OS?

| Layer          | Name            | Purpose                                      |
|----------------|-----------------|----------------------------------------------|
| **Network**    | DeepShitNet     | The hidden mesh/overlay (Reticulum + LoRa + whatever works) |
| **Environment**| DeepShit OS     | The actual operating system and services you use |

Everything in DeepShit OS is only accessible from inside DeepShitNet.

## Core Principles

- Uses **all available infrastructure** but never depends on it.
- **Cryptographic identities** instead of accounts.
- **No one controls it** — not even the creators.
- Starts practical (house + Raspberry Pi 5) but designed to scale.
- Heavy emphasis on **branding and humour** (Steve, `.deep` files, seagull wallpaper, etc.).

## Current Status (v0.1)

- Base: Reticulum (cryptographic networking stack) running on Raspberry Pi 5
- Starting scope: Single house + all devices inside it
- First focus: Visuals & naming only (no menu changes yet)
- Default wallpaper: A seagull aggressively eating the cables of a rusty satellite dish lying on the ground
- Branding elements: Steve references, `.deep` file extensions/naming

## Branding & In-Jokes

- **Steve** — The critical (and fragile) infrastructure node. The system boots by "searching for Steve".
- **.deep** — Everything gets a `.deep` extension or naming (files, versions, messages).
- **Seagull** — Official mascot. The default wallpaper shows one trying to destroy a satellite dish.
- UK original, built with maximum British chaos.

## Tech Foundation (Planned)

- **Networking**: Reticulum (primary) + LoRa/Meshtastic for true off-grid capability
- **Base OS**: Customised Linux (starting with visuals and names on Parrot OS or Raspberry Pi OS)
- **Access**: Sideband app + local clients (no web dashboard as main interface)

## Why?

Because normal internet is fragile, centralised, and boring.
We want something that keeps working when everything else dies, looks ridiculous while doing it, and belongs to no one.
